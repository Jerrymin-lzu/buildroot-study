# 4. 使用 v4l2loopback 模拟摄像头

本文说明如何在 ARM 虚拟机中通过 `v4l2loopback` 内核模块创建虚拟视频设备，模拟摄像头工作，并使用工具观测 V4L2 子系统的软件栈。

---

## 背景

### 什么是 v4l2loopback

`v4l2loopback` 是一个内核模块，可创建"虚拟视频设备"（`/dev/video*`）。上层 V4L2 应用会将其视为普通摄像头，但视频数据并非来自真实硬件，而是由另一个用户态程序"喂入"。

```
┌───────────────────────────────────────────────────────────┐
│  VM (ARM aarch64)                                         │
│                                                           │
│  ┌──────────────┐     ┌──────────────┐                    │
│  │ 视频生产者     │────>│ /dev/video0  │<── v4l2loopback   │
│  │ (ffmpeg/     │ 写入 │ (虚拟设备)    │    内核模块        │
│  │  gstreamer/  │      └──────┬───────┘                    │
│  │  自定义程序)   │            │                            │
│  └──────────────┘     ┌──────┴───────┐                    │
│                        │ 视频消费者     │                    │
│                        │ (v4l2-ctl/   │                    │
│                        │  ffmpeg/     │                    │
│                        │  opencv/     │                    │
│                        │  浏览器等)    │                    │
│                        └──────────────┘                    │
└───────────────────────────────────────────────────────────┘
```

### 本项目的 V4L2 相关组件

Buildroot 2023.11.1 已自带以下相关包：

| 包名 | 作用 | 层级 |
|------|------|------|
| `v4l2loopback` | 虚拟视频设备内核模块 | 内核 |
| `libv4l` | V4L2 用户态抽象库（含格式转换） | 用户态库 |
| `v4l-utils`（libv4l 子选项） | 观测工具集（v4l2-ctl、media-ctl、v4l2-compliance） | 用户态工具 |
| `yavta` | Yet Another V4L2 Test Application（轻量测试工具） | 用户态工具 |
| `v4l2grab` | 从 V4L2 设备抓取 JPEG 帧 | 用户态工具 |
| `ffmpeg` | 多媒体框架（可充当视频生产者/消费者） | 用户态工具 |
| `gstreamer1` | GStreamer 多媒体框架（含 V4L2 插件） | 用户态工具 |

## 步骤 1：启用内核 V4L2 支持

### 1.1 当前状态

```bash
grep CONFIG_MEDIA_SUPPORT buildroot-2023.11.1/output/build/linux-6.1.44/.config
```

当前输出为 `# CONFIG_MEDIA_SUPPORT is not set`，说明 V4L2 子系统尚未启用。

### 1.2 配置内核

进入 kernel menuconfig 启用 V4L2 框架：

```bash
cd buildroot-2023.11.1
make linux-menuconfig
```

需要启用的内核选项：

```
Device Drivers
  └── <*> Multimedia support  [CONFIG_MEDIA_SUPPORT]
        └── Media device types  --->
              └── [*] Cameras and video grabbers  [CONFIG_MEDIA_CAMERA_SUPPORT]
        └── Media core support
              └── [*] Video4Linux2 core  [CONFIG_VIDEO_DEV]
        └── Media drivers
              └── [*] Media USB Adapters  (如需要 USB 摄像头)
        └── Media controller API  (可选，用于复杂拓扑)
              └── [*] Media Controller API  [CONFIG_MEDIA_CONTROLLER]
        └── [*] V4L2 sub-device userspace API  [CONFIG_V4L2_SUBDEV_API]  (可选)
```

**最小必需选项**（v4l2loopback 编译必需的）：

| Kconfig 选项 | 含义 |
|-------------|------|
| `CONFIG_MEDIA_SUPPORT=y` | 多媒体子系统总开关 |
| `CONFIG_VIDEO_DEV=y` | V4L2 核心框架 |
| `CONFIG_I2C=y` | Linux 5.18+ 后 VIDEO_DEV 依赖 I2C |

> **提示**：如果只通过 menuconfig 勾选 v4l2loopback 包，Buildroot 的 `LINUX_CONFIG_FIXUPS` 机制会自动打开以上三个选项。手动配置内核可启用更多功能（如完整 media controller）。

## 步骤 4：启动虚拟机并加载模块

### 4.1 加载 v4l2loopback

在虚拟机中：

```bash
# 先加载依赖
modprobe videodev

# 加载 v4l2loopback，创建 2 个虚拟视频设备
modprobe v4l2loopback devices=2 video_nr=10,11 exclusive_caps=1

# 或者用 insmod
insmod /lib/modules/6.1.44/extra/v4l2loopback.ko devices=2
```

模块参数：

| 参数 | 含义 | 示例 |
|------|------|------|
| `devices` | 创建的虚拟设备数量 | `devices=2` |
| `video_nr` | 指定 /dev/video 编号 | `video_nr=10,11` |
| `card_label` | 设备标签（显示在应用列表中） | `card_label="Virtual Cam"` |
| `exclusive_caps` | 报告独占能力标志（兼容 Chrome/火狐等） | `exclusive_caps=1` |
| `max_buffers` | 最大缓冲区数量 | `max_buffers=4` |

### 4.2 验证模块已加载

```bash
dmesg | grep -i v4l2loopback
# [  123.456] v4l2loopback: loading out-of-tree module taints kernel.
# [  123.457] v4l2loopback: module loaded, devices=2, ...

ls /dev/video*
# /dev/video10  /dev/video11
```

## 步骤 5：观测 V4L2 软件栈

以下工具构成完整的 V4L2 观测链。

### 5.1 v4l2-ctl — 设备信息与格式查询

```bash
# 列出所有 V4L2 设备
v4l2-ctl --list-devices
# 输出:
#   v4l2loopback (platform:v4l2loopback-000):
#       /dev/video10
#       /dev/video11

# 查看设备支持的所有格式
v4l2-ctl -d /dev/video10 --all
# 输出: Driver Info, Format, Selection, Codec 等全部信息

# 查看当前视频格式
v4l2-ctl -d /dev/video10 --get-fmt-video
# 输出: Format Video Capture: Width/Height, Pixel Format, Field, ...

# 设置视频格式（在喂视频之前）
v4l2-ctl -d /dev/video10 --set-fmt-video=width=640,height=480,pixelformat=YUYV
```

### 5.2 media-ctl — 媒体控制器拓扑

```bash
# 查看 media 设备拓扑（适用于复杂 pipeline）
media-ctl -p
# 打印完整的设备拓扑图
# 每个 entity（实体）、pad（端口）、link（连接）都会列出
```

### 5.3 v4l2-compliance — 兼容性检查

```bash
# 对虚拟设备运行 V4L2 合规性测试
v4l2-compliance -d /dev/video10
# 逐项检查: open/close, ioctls, formats, buffer operations, ...
# 输出通过/失败/警告
```

### 5.4 yavta — 轻量测试

```bash
# 查看设备能力
yavta -l /dev/video10

# 捕获一帧并保存到文件
yavta -c1 -Foutput.yuv /dev/video10
```

### 5.5 内核调试接口

```bash
# V4L2 设备 sysfs 信息
ls /sys/class/video4linux/
cat /sys/class/video4linux/video10/name
cat /sys/class/video4linux/video10/dev

# V4L2 设备 debug 接口（需内核开启 CONFIG_DEBUG_FS）
mount -t debugfs none /sys/kernel/debug
cat /sys/kernel/debug/v4l2loopback/video10/format   # 查看当前格式
cat /sys/kernel/debug/v4l2loopback/video10/fps       # 查看帧率
```

## 步骤 6：模拟实际摄像头工作

### 6.1 用 ffmpeg 喂入视频流（模拟摄像头输出）

在虚拟机中：

```bash
# 方式 A：喂入静态图片（循环作为视频流）
ffmpeg -loop 1 -i /path/to/image.jpg \
    -f v4l2 -pix_fmt yuyv422 \
    /dev/video10

# 方式 B：喂入视频文件（模拟录像回放）
ffmpeg -re -i /path/to/video.mp4 \
    -f v4l2 -pix_fmt yuyv422 \
    /dev/video10

# 方式 C：生成测试图案（纯色/彩条）
ffmpeg -f lavfi -i testsrc=size=640x480:rate=30 \
    -f v4l2 -pix_fmt yuyv422 \
    /dev/video10

# 方式 D：生成 SMPTE 彩条（标准测试图）
ffmpeg -f lavfi -i smptebars=size=640x480:rate=30 \
    -f v4l2 -pix_fmt yuyv422 \
    /dev/video10
```

### 6.2 用 GStreamer 喂入视频流

```bash
# 生成测试图案
gst-launch-1.0 videotestsrc ! \
    video/x-raw,format=YUY2,width=640,height=480,framerate=30/1 ! \
    v4l2sink device=/dev/video10
```

### 6.3 另一个进程读取虚拟摄像头

```bash
# 在另一个终端中，用 ffmpeg 从虚拟设备读取
ffmpeg -f v4l2 -i /dev/video10 -vframes 1 capture.jpg

# 或用 v4l2-ctl 捕获一帧
v4l2-ctl -d /dev/video10 --stream-mmap --stream-count=1 --stream-to=frame.raw
```

## 完整验证流程

```
虚拟机中执行：

# 1. 加载模块
modprobe v4l2loopback devices=1 video_nr=10 exclusive_caps=1
ls /dev/video10                                   # 确认设备存在

# 2. 观测设备信息
v4l2-ctl -d /dev/video10 --all                    # 查看能力
v4l2-ctl -d /dev/video10 --get-fmt-video          # 查看当前格式

# 3. 设置格式
v4l2-ctl -d /dev/video10 --set-fmt-video=width=640,height=480,pixelformat=YUYV

# 4. 喂入视频（后台运行）
ffmpeg -f lavfi -i testsrc=size=640x480:rate=30 \
    -f v4l2 /dev/video10 &
PID=$!

# 5. 观测数据流
# 终端 A：查看 v4l2-ctl 流统计
v4l2-ctl -d /dev/video10 --stream-mmap --stream-count=10

# 终端 B：实时查看队列状态
cat /sys/kernel/debug/v4l2loopback/video10/format
cat /sys/kernel/debug/v4l2loopback/video10/fps

# 终端 C: 用 ffmpeg 抓取一帧
ffmpeg -f v4l2 -i /dev/video10 -vframes 1 output.jpg

# 6. 停止
kill $PID
```

## V4L2 软件栈观测节点

```
┌────────────────────────────────────────────────────┐
│  用户态                                             │
│                                                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐         │
│  │ v4l2-ctl │  │  yavta   │  │  ffmpeg  │  ...    │
│  │ --all    │  │  -l      │  │  -f v4l2 │         │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘         │
│       │              │              │               │
│       └──────────────┼──────────────┘               │
│                      │ ioctl()                      │
├──────────────────────┼──────────────────────────────┤
│  sysfs/debugfs       │  内核态                      │
│                      ▼                              │
│  /sys/class/     ┌─────────────┐                    │
│  video4linux/    │  V4L2 Core  │  CONFIG_VIDEO_DEV  │
│                  └──────┬──────┘                    │
│  /sys/kernel/           │                           │
│  debug/          ┌──────┴───────────┐               │
│  v4l2loopback/   │  v4l2loopback    │  .ko 模块     │
│                  └──────┬───────────┘               │
│                         │                           │
│              ┌──────────┴───────────┐               │
│              │ /dev/video10         │               │
│              │ /dev/video11         │               │
│              └──────────────────────┘               │
└────────────────────────────────────────────────────┘
```

| 观测点 | 工具/接口 | 可看到的信息 |
|--------|----------|------------|
| 设备列表 | `v4l2-ctl --list-devices` | 所有 V4L2 设备名和节点 |
| 设备能力 | `v4l2-ctl -d /dev/videoX --all` | 驱动名、支持格式、控件、输入输出 |
| 当前格式 | `v4l2-ctl --get-fmt-video` | 分辨率、像素格式、帧率 |
| 流状态 | `v4l2-ctl --stream-mmap --stream-count=N` | 实际抓帧数、丢帧数 |
| 设备拓扑 | `media-ctl -p` | entity → pad → link 完整拓扑 |
| 合规性 | `v4l2-compliance -d /dev/videoX` | 各 ioctl 支持情况、是否合规 |
| sysfs | `cat /sys/class/video4linux/videoX/name` | 设备名称、设备号 |
| debugfs | `cat /sys/kernel/debug/v4l2loopback/videoX/*` | 格式、FPS、队列深度 |
| 内核日志 | `dmesg \| grep v4l` | 模块加载、错误、警告 |

## 常见问题

**Q1: modprobe 报 `Module not found`？**

先执行 `depmod -a` 刷新模块依赖数据库。

**Q2: v4l2loopback 加载失败，dmesg 显示 `Unknown symbol`？**

内核未启用 V4L2 框架。确认内核 `.config` 中：
```bash
grep "CONFIG_MEDIA_SUPPORT\|CONFIG_VIDEO_DEV" output/build/linux-6.1.44/.config
```
应均为 `=y`。

**Q3: `/dev/video10` 存在但 ffmpeg 无法写入？**

检查 v4l2loopback 是否以可写模式创建：
```bash
# 查看是否已有消费者在读取（exclusive_caps=1 时可能冲突）
fuser /dev/video10
```

**Q4: 抓取的帧是黑屏/绿屏？**

说明视频生产者未在喂数据，或格式不匹配。先用 `v4l2-ctl --set-fmt-video` 设置正确格式，再喂视频。

**Q5: v4l2-ctl 命令不存在？**

确认已勾选 `libv4l → v4l-utils tools` 并重新 `make`。

**Q6: 如何模拟多个不同分辨率的摄像头？**

```bash
modprobe v4l2loopback devices=3 \
    video_nr=10,11,12 \
    card_label="Cam 640x480","Cam 1280x720","Cam 1920x1080"

# 然后分别设置每个设备的格式
v4l2-ctl -d /dev/video10 --set-fmt-video=width=640,height=480,pixelformat=YUYV
v4l2-ctl -d /dev/video11 --set-fmt-video=width=1280,height=720,pixelformat=YUYV
v4l2-ctl -d /dev/video12 --set-fmt-video=width=1920,height=1080,pixelformat=YUYV
```
